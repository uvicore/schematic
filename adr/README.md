# Architecture Decision Records (ADRs)

One file per **significant, hard-to-reverse decision** about the scaffold/installer itself — its
layout, the generators, default conventions baked into generated apps. Routine work goes in the
[journal](../journal/README.md).

Framework *behavior* decisions belong in the framework repo's ADRs — link to them rather than
duplicating.

- **Filename**: `NNNN-kebab-title.md` (sequential).
- **Format** (compact Nygard): Status, Date, Context, Decision, Consequences.
- **Supersede, don't rewrite**: mark a replaced ADR `Superseded by ADR-XXXX`.

## Template

```markdown
# ADR NNNN: <Title>

- **Status:** Accepted | Proposed | Superseded by ADR-XXXX
- **Date:** YYYY-MM-DD

## Context
## Decision
## Consequences
```
