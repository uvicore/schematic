# Journal — Schematic

A dated, chronological log of substantive work in **this repo** (the app scaffold / installer
template). Records *what changed and why* behind a scaffold or skill change, not just the diff.

- **One file per day**: `journal/YYYY-MM-DD.md`. Append each piece of work as an `##` (H2) entry.
- **Update at the end of any substantive task** (stub/scaffold change, generator change, skill
  update). Skip trivia.
- **Cross-repo work**: scaffold changes usually follow a `framework` change — note it under
  **Related** so the lineage is clear.

## Entry template

```markdown
## <Short title>
- **Intent:** what was asked / the goal
- **Changed:** what changed and *why*
- **Files:** key files touched
- **Related:** framework/docs edits, ADRs, follow-ups
```

Significant decisions get an [ADR](../adr/README.md).
